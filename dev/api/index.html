API

Results.jl is structured into four modules:

  • Results.Types defines the core types
  • Results.Functions defines functions for operating on Results and Options
  • Results.Macros defines macros for working with result types
  • Results.Collection defines Option interfaces for working with collections
  • Results.Operators defines operators for working with result types.

Important members of each module are re-exported, so the end user usually does not need to import these modules directly. Operators are not exported by default.
